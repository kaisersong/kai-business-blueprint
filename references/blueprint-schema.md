# Blueprint Schema Reference

## Top-level structure

```json
{
  "version": "1.0",
  "meta": {},
  "context": {},
  "library": {},
  "relations": [],
  "views": [],
  "editor": {},
  "artifacts": {}
}
```

## meta

```json
{
  "title": "string — blueprint title",
  "industry": "common | finance | manufacturing | retail",
  "blueprintType": "architecture | domain-knowledge",
  "revisionId": "rev-YYYYMMDD-NN",
  "parentRevisionId": "null | previous revisionId",
  "lastModifiedAt": "ISO8601",
  "lastModifiedBy": "ai | human"
}
```

- `blueprintType` is set by the seed template, not by user input.
- `architecture` → fill `library.capabilities/actors/flowSteps/systems`.
- `domain-knowledge` → fill `library.knowledge.*`.

## context

```json
{
  "goals": ["string"],
  "scope": ["string"],
  "assumptions": ["string"],
  "constraints": ["string"],
  "sourceRefs": [{"type": "inline-text", "excerpt": "..."}],
  "clarifyRequests": [],
  "clarifications": []
}
```

## library

Entity collections:

| Collection | blueprintType | Description |
|------------|---------------|-------------|
| `capabilities` | architecture | Business capability areas |
| `actors` | architecture | Roles / people involved |
| `flowSteps` | architecture | Business process steps |
| `systems` | architecture | IT systems supporting capabilities |
| `knowledge` | domain-knowledge | See `knowledge-entities-schema.md` |

### capabilities

```json
{"id": "cap-xxx", "name": "...", "level": 1, "description": "...", "ownerActorIds": [], "supportingSystemIds": []}
```

Field definitions: `references/entities-schema.md`.

### actors

```json
{"id": "actor-xxx", "name": "..."}
```

### flowSteps

```json
{"id": "flow-xxx", "name": "...", "actorId": "actor-xxx", "capabilityIds": ["cap-xxx"], "systemIds": [], "stepType": "task", "inputRefs": [], "outputRefs": []}
```

### systems

```json
{"id": "sys-xxx", "kind": "system", "name": "...", "aliases": [], "description": "...", "resolution": {"status": "canonical", "canonicalName": "..."}, "capabilityIds": ["cap-xxx"]}
```

Field definitions and layer rules: `references/systems-schema.md`.

## relations

```json
[{"id": "rel-xxx", "type": "supports", "from": "sys-xxx", "to": "cap-xxx", "label": "支撑"}]
```

Common relation types for architecture blueprints: `supports`, `depends_on`, `feeds_into`, `communicates_with`.
Knowledge-specific relation types: see `references/knowledge-entities-schema.md` → Relations section.

## views

Leave as `[]`. Views are generated by export tools, not authored manually.

## editor

```json
{"fieldLocks": {}, "theme": "enterprise-default"}
```

See `references/authoring-rules.md` for field lock behavior.

## artifacts

Leave as `{}`. Populated by export tools.
